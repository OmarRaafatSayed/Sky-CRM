# نظام الكوبونات (Coupon System)

## نظرة عامة
نظام شامل لإدارة كوبونات الخصم يسمح للمديرين بإنشاء وإدارة كوبونات الخصم، وللمستخدمين بتطبيقها أثناء عملية الدفع.

## الميزات الرئيسية

### 1. إدارة الكوبونات
- إنشاء كوبونات جديدة مع خيارات متقدمة
- تعديل وحذف الكوبونات الموجودة
- تفعيل/إيقاف الكوبونات
- توليد أكواد عشوائية للكوبونات

### 2. أنواع الخصم
- **خصم نسبة مئوية**: خصم بنسبة معينة من المبلغ الإجمالي
- **خصم مبلغ ثابت**: خصم مبلغ محدد من المبلغ الإجمالي

### 3. قيود الاستخدام
- الحد الأدنى لمبلغ الطلب
- الحد الأقصى لمبلغ الخصم (للنسبة المئوية)
- عدد مرات الاستخدام المحدود
- تاريخ البدء والانتهاء
- تطبيق على منتجات/خدمات محددة
- تحديد المستخدمين المسموح لهم

### 4. تتبع الاستخدام
- سجل كامل لاستخدام الكوبونات
- تتبع المستخدم وتاريخ الاستخدام
- حساب المبلغ الموفر
- ربط بمعرف الطلب

## هيكل قاعدة البيانات

### جدول `coupons`
```sql
- id: UUID (Primary Key)
- code: VARCHAR(50) UNIQUE - كود الكوبون
- name: VARCHAR(255) - اسم الكوبون
- description: TEXT - وصف الكوبون
- discount_type: ENUM('percentage', 'fixed') - نوع الخصم
- discount_value: DECIMAL(10,2) - قيمة الخصم
- max_discount_amount: DECIMAL(10,2) - الحد الأقصى للخصم
- min_order_amount: DECIMAL(10,2) - الحد الأدنى للطلب
- usage_limit: INTEGER - عدد مرات الاستخدام المسموح
- used_count: INTEGER - عدد مرات الاستخدام الحالي
- is_active: BOOLEAN - حالة الكوبون
- valid_from: TIMESTAMP - تاريخ البدء
- valid_until: TIMESTAMP - تاريخ الانتهاء
- applicable_products: TEXT[] - المنتجات المطبقة عليها
- allowed_users: TEXT[] - المستخدمين المسموح لهم
- organization_id: UUID - معرف المؤسسة
- created_by: UUID - منشئ الكوبون
- created_at: TIMESTAMP - تاريخ الإنشاء
- updated_at: TIMESTAMP - تاريخ التحديث
```

### جدول `coupon_usage`
```sql
- id: UUID (Primary Key)
- coupon_id: UUID - معرف الكوبون
- user_id: UUID - معرف المستخدم
- order_id: VARCHAR(255) - معرف الطلب
- discount_amount: DECIMAL(10,2) - مبلغ الخصم المطبق
- original_amount: DECIMAL(10,2) - المبلغ الأصلي
- final_amount: DECIMAL(10,2) - المبلغ النهائي
- used_at: TIMESTAMP - تاريخ الاستخدام
- organization_id: UUID - معرف المؤسسة
```

## المكونات (Components)

### 1. CouponsManagement
مكون إدارة الكوبونات الرئيسي يوفر:
- واجهة لعرض جميع الكوبونات
- نموذج إنشاء/تعديل الكوبونات
- إجراءات الحذف والتفعيل/الإيقاف
- توليد أكواد عشوائية

**الموقع**: `src/components/CouponsManagement.tsx`

### 2. CouponInput
مكون إدخال الكوبون في صفحة الدفع يوفر:
- حقل إدخال كود الكوبون
- التحقق من صحة الكوبون
- عرض تفاصيل الخصم المطبق
- إزالة الكوبون المطبق

**الموقع**: `src/components/CouponInput.tsx`

### 3. useCoupon Hook
Hook مخصص لإدارة حالة الكوبون يوفر:
- تطبيق الكوبون
- إزالة الكوبون
- تسجيل استخدام الكوبون
- حساب المبلغ النهائي

## التكامل مع نظام الدفع

### في صفحة الدفع (Payment.tsx)
1. إضافة مكون `CouponInput`
2. استخدام `useCoupon` hook
3. تمرير المبلغ النهائي لنظام الدفع
4. تسجيل استخدام الكوبون عند نجاح الدفع

### تدفق العمل
1. المستخدم يدخل كود الكوبون
2. التحقق من صحة الكوبون وشروطه
3. حساب مبلغ الخصم والمبلغ النهائي
4. عرض تفاصيل الخصم للمستخدم
5. تمرير المبلغ النهائي لنظام الدفع
6. تسجيل استخدام الكوبون عند نجاح الدفع

## الأمان والصلاحيات

### Row Level Security (RLS)
- المستخدمون يمكنهم رؤية كوبونات مؤسستهم فقط
- المديرون فقط يمكنهم إنشاء وتعديل الكوبونات
- تسجيل الاستخدام محمي بصلاحيات المؤسسة

### التحقق من الصحة
- التحقق من صلاحية الكوبون قبل التطبيق
- التحقق من عدد مرات الاستخدام
- التحقق من الحد الأدنى للطلب
- التحقق من تاريخ الصلاحية

## الاستخدام

### للمديرين
1. الانتقال إلى صفحة إدارة الكوبونات
2. إنشاء كوبون جديد بالشروط المطلوبة
3. تفعيل الكوبون
4. مراقبة استخدام الكوبون

### للمستخدمين
1. في صفحة الدفع، إدخال كود الكوبون
2. التحقق من تطبيق الخصم
3. إكمال عملية الدفع بالمبلغ المخفض

## التحسينات المستقبلية
- كوبونات خاصة بمنتجات محددة
- كوبونات خاصة بفئات مستخدمين
- كوبونات تراكمية
- إشعارات انتهاء صلاحية الكوبونات
- تقارير تفصيلية عن استخدام الكوبونات
- كوبونات ديناميكية بناءً على سلوك المستخدم
